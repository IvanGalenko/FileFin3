# FileFin3

Итак, по пунктам:
1. Методы:
	DirFileSize рекурсивно считывает размер ФАЙЛОВ в папках и тем самым выдает размер папки. 
		Размер папки = размер вложенных файлов. Итого мы получает размер папки
	DirFileDelete удаляет файлы и папки, подходящие по времени. 
		При этом еще и пополняет счетчики размера удаленных файлов и количества самих файлов
2. Логика:
	Проверяется исходный размер
	Запускается удаление. Удаление рекурсивно считывает, сверяет и удаляет
	Запускается проверка конечного размера
3. Моменты:
	Пришлось долго шаманить, чтобы понять как правильно удалять папки, ибо:
		1. Если в папку вложена папка, то всё идет боком. Изменение вложенной папки != изменение основной папки
		2. Не всегда изменение файла в папке изменяет саму папку. Закономерность не уловил.

Но в целом, путем попыток и сверки конечных сумм удалось проверить работоспособность на объективном примере:
	1. Папка с вложенным файлом. Оба изменяются в момент А
	2. Папка с вложенным файлов. Оба изменяются в момент Б
	3. Файл. Изменяется в момент А
	Если запустить программу сразу после момента Б, чтобы А являлся подходящим условием (как и условие, собственно), то всё работает

	Для удобства, все анализы файлов и папок выводятся в консоль для сверки
